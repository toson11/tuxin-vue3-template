## set/unset命令

- 显示所有环境变量 `set`
- 设置shell选项 `set [options] [argument]`
  - -e：errexit 如果任何语句的执行结果不是true则退出。
  - -u：当执行时使用未定义的变量，则显示错误信息并退出。
  - -x：执行命令时把命令和其参数显示出来。
  - +x：禁止-x选项，即禁止显示命令和参数。
  - -o pipefail：如果管道命令中的任何一个子命令失败则整个管道命令返回失败。
  - ...
- 设置环境变量 `set VARNAME=value`
- 删除环境变量 `unset VARNAME`

## export导出变量

与set不同，export命令会将变量导出到子进程中，使得子进程可以通过$VARNAME访问到该变量。

- 设置环境变量 `export VARNAME=value`

## source命令

- 读取并在**当前shell进程**执行文件中的命令。比如执行激活虚拟环境的脚本文件 `source venv/bin/activate`
- 与./不同的是，./会在**子shell进程**中执行脚本文件中的命令。./执行的脚本文件中的变量在脚本执行完后无法在当前shell进程中访问到。

## 设置别名

- alias 设置命令别名 `alias alias_name='command'`
- unalias 删除命令别名 `unalias alias_name`

## 变量声明与使用

- 声明变量 `VARNAME=value`
- 使用变量 `echo $VARNAME`
- 设置默认值 `${VARNAME:-default}`：如果VARNAME未定义或为空，则返回default，否则返回VARNAME的值。
- 设置默认值并赋值 `${VARNAME:=default}`：如果VARNAME未定义或为空，则将VARNAME的值设为default，并返回default，否则返回VARNAME的值。
- 变量为空报错并退出 `${VARNAME:?message}`：如果VARNAME未定义或为空，则输出message并退出。

# 查看端口

- 查看端口占用 `sudo lsof -i -P -n | grep LISTEN`
- 查看端口：netstat -anp | grep 8080,a指all，n指numeric以数字形式显示地址和端口号，p指process，显示进程信息
- 查看端口占用 `lsof -i:端口号`
- 关闭端口 `kill 进程号` 或 `sudo kill -9 进程号`
